# 추천 기준

추천 API는 저장된 HS Portal 비교과 프로그램 목록과 사용자의 수업 시간표를 비교하여 각 프로그램을 `참여 가능`, `확인 필요`, `참여 불가`로 분류합니다. 단순히 프로그램을 제외하는 방식이 아니라, 일정 정보가 부족하거나 애매한 프로그램은 `확인 필요`로 남겨 사용자가 상세 페이지에서 판단할 수 있게 합니다.

## 입력 데이터

추천 계산에는 다음 데이터가 사용됩니다.

| 데이터 | 설명 |
| --- | --- |
| 수업 시간표 | LLM이 추출하거나 사용자가 편집한 `course_name`, `day_of_week`, `start_time`, `end_time` |
| 프로그램 일정 | HS Portal 목록/상세 페이지에서 수집한 `schedules` |
| 신청 상태 | 접수중, 접수대기, 접수예정 등 상태 코드와 접수 가능 여부 |
| 신청 인원 | 현재 신청 인원, 정원, 무제한 여부 |
| 비교과 포인트 | HS Portal에서 제공하는 포인트 값 |
| 분류 정보 | HS Portal 상세검색 기준의 대분류/세부 분류 |

## 일정 유형 분류

각 프로그램은 먼저 운영 일정 형태에 따라 `schedule_kind`로 분류됩니다.

| 값 | 기준 | 추천 판단 |
| --- | --- | --- |
| `same_day` | 각 일정의 시작일과 종료일이 같은 날짜 | 수업 시간과 직접 비교 |
| `short_period` | 여러 날짜에 걸치지만 7일 미만 | 회차별 실제 참석 시간 확인 필요 |
| `long_period` | 하나 이상의 일정 기간이 7일 이상 | 장기/온라인/상시 활동 가능성이 있어 확인 필요 |
| `no_schedule` | 비교할 일정 정보 없음 | 상세 페이지 확인 필요 |
| `invalid_schedule` | 시작/종료 값을 파싱할 수 없거나 종료가 시작보다 빠름 | 데이터 확인 필요 |

## 수업 시간 충돌 판단

`same_day` 일정만 수업 시간과 직접 비교합니다. 프로그램 시작/종료 시각과 같은 요일의 수업 시작/종료 시각을 분 단위로 변환한 뒤, 두 구간이 겹치는지 확인합니다.

두 시간 구간 `A`, `B`는 다음 조건을 만족하면 충돌로 봅니다.

```text
A.start < B.end and B.start < A.end
```

충돌이 발생하면 `conflicts` 배열에 수업명, 수업 시간, 프로그램 시간을 기록합니다.

## 참여 가능 여부 분류

| 상태 | 기준 |
| --- | --- |
| `available` | 당일 종료 일정이고 수업 시간과 겹치지 않으며 접수 가능성이 높음 |
| `needs_review` | 일정이 기간형이거나, 일정 정보가 없거나, 접수 상태가 명확하지 않거나, 데이터 확인이 필요한 경우 |
| `unavailable` | 정원이 마감되었거나 수업 시간과 직접 충돌하는 경우 |

## 세부 판단 순서

1. 정원이 마감되었는지 먼저 확인합니다.
2. 마감된 프로그램은 일정과 관계없이 `unavailable`로 분류합니다.
3. 일정 유형이 `same_day`이고 수업과 겹치면 `unavailable`로 분류합니다.
4. 일정 유형이 `same_day`이고 수업과 겹치지 않으면 접수 가능 상태를 확인합니다.
5. 접수 가능성이 있으면 `available`로 분류합니다.
6. 접수 상태가 불명확하거나 접수 가능 상태가 아니면 `needs_review`로 분류합니다.
7. `short_period`, `long_period`, `no_schedule`, `invalid_schedule`은 시간표만으로 확정하지 않고 `needs_review`로 분류합니다.

## 신뢰도

추천 결과에는 판단 신뢰도인 `confidence`가 포함됩니다.

| 값 | 의미 |
| --- | --- |
| `high` | 당일 일정이어서 시간표와 직접 비교했거나 정원 마감처럼 명확한 기준이 있음 |
| `medium` | 기간형 일정이라 추가 확인이 필요하지만 일정 범위는 해석 가능함 |
| `low` | 일정이 없거나 일정 값이 잘못되어 판단 근거가 약함 |

## 경고 메시지

프로그램별 `warnings`에는 사용자가 상세 페이지에서 확인해야 하는 내용이 들어갑니다.

대표적인 경고는 다음과 같습니다.

- 현재 신청 인원이 정원 이상입니다.
- 현재 상태가 접수 가능 상태가 아닙니다.
- 등록된 시작일과 종료일만으로 매일 참석 여부를 확정할 수 없습니다.
- 교육기간 전체가 실제 참석 시간을 의미하지 않을 수 있습니다.
- 프로그램 상세 페이지에서 실제 운영 일정을 확인해야 합니다.
- 저장된 운영 일정의 시작/종료 값을 해석할 수 없습니다.
- HS Portal 상세검색의 알려진 세부 분류와 일치하지 않습니다.

## 점수 계산

추천 점수는 가용성, 판단 신뢰도, 일정 유형, 비교과 포인트, 충돌 수를 반영합니다.

```text
score =
  availability_score
  + confidence_score
  + schedule_kind_score
  + point_score
  - conflict_penalty
```

### 가용성 점수

| 상태 | 점수 |
| --- | --- |
| `available` | 100 |
| `needs_review` | 60 |
| `unavailable` | 0 |

### 신뢰도 점수

| 신뢰도 | 점수 |
| --- | --- |
| `high` | 15 |
| `medium` | 8 |
| `low` | 0 |

### 일정 유형 점수

| 일정 유형 | 점수 |
| --- | --- |
| `same_day` | 18 |
| `short_period` | 8 |
| `long_period` | 2 |
| `no_schedule` | 0 |
| `invalid_schedule` | 0 |

### 비교과 포인트 점수

비교과 포인트는 최대 30점까지만 반영하며, 3으로 나눈 값을 점수에 더합니다.

```text
point_score = min(points, 30) / 3
```

### 충돌 감점

수업과 충돌한 일정 1개당 20점을 감점합니다.

```text
conflict_penalty = conflict_count * 20
```

최종 점수는 0점 미만으로 내려가지 않도록 보정합니다.

## 정렬 기준

추천 결과는 다음 순서로 정렬됩니다.

1. `available`
2. `needs_review`
3. `unavailable`

같은 상태 안에서는 점수가 높은 순서로 정렬하고, 점수도 같으면 프로그램 제목순으로 정렬합니다.
